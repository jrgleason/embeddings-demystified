# Vector Databases and Embeddings Demystified

A comprehensive 2-hour technical presentation for CodeMash 2025 that demystifies vector embeddings and similarity search - the technologies powering Netflix recommendations, Google Photos search, and ChatGPT document Q&A.

**Presenter:** Jackie Gleason
**Duration:** 2 hours (including demos)
**Event:** CodeMash 2025

## What You'll Learn

1. **What are Vectors and Embeddings** - Understanding numerical representations of meaning
2. **How to Use Embeddings** - Practical implementation with Python and JavaScript
3. **What Problems They Solve** - Real-world applications from recommendations to RAG
4. **Vector Database Architecture** - HNSW, IVF, Product Quantization algorithms
5. **Production Considerations** - Scaling, performance, and choosing the right tools

## Prerequisites

- Basic Python knowledge (no ML background required)
- Node.js 18+ (for the interactive demo)
- Ollama running locally (for embeddings)

## Quick Start - Interactive Demo

The `my-shot` demo provides an interactive way to explore vector embeddings:

```bash
cd demos/my-shot
npm install
npm run dev
```

Then visit http://localhost:3000 to explore:

- **Vector Store** - Add prompts, generate embeddings, and search for similar content
- **t-SNE Visualization** - See how words cluster in 2D space based on meaning
- **Embedding Display** - Examine raw embedding vectors and their statistics

### Demo Requirements

Make sure Ollama is running with the embedding model:

```bash
ollama pull qwen3-embedding:0.6b
ollama serve
```

## Key Concepts Covered

### Vector Embeddings
- Numerical representations that capture semantic meaning
- Generated by AI models (OpenAI, Sentence Transformers, Qwen)
- Typical dimensions: 384, 768, 1024, or 1536

### Similarity Metrics
| Metric | Range | Best For |
|--------|-------|----------|
| Cosine Similarity | -1 to 1 | Text embeddings (default choice) |
| Euclidean Distance | 0 to ∞ | When magnitude matters |
| Dot Product | -∞ to ∞ | Speed-critical, normalized vectors |

### ANN Algorithms
| Algorithm | Speed | Memory | Accuracy | Best Scale |
|-----------|-------|--------|----------|------------|
| HNSW | O(log n) | High | 95-99% | <100M |
| IVF | O(n/c) | Medium | 90-95% | 10M-500M |
| PQ | Fast | Low | 85-90% | Any |
| IVF + PQ | Fast | Low | 90-95% | 100M+ |

### Vector Storage Options
- **Pinecone, Milvus** - Vector-native databases (fast ANN indexing)
- **LangChainJS, Redis** - In-memory stores (low-latency, ephemeral)
- **MongoDB Atlas** - Document DB with vector search
- **Neo4j** - Graph DB with embeddings for relational queries
