# Vs SQL, Etc

## Database Paradigms Compared

### Different Tools for Different Jobs:

**SQL:** Use complex logic to get me exactly what I am looking for and all related information.
- Precise queries with joins
- Structured data with relationships

**NoSQL:** Use an ID to get a particular record
- Fast key-value lookups
- Flexible schema

**Graph:** Get me information related to a particular record
- Navigate relationships
- Connected data exploration

**Vector:** Get me the records similar to mine
- Semantic similarity search
- Find "things like this"

---

**Notes:**
- Clear comparison of database paradigms
- Each serves different use cases
- Vector databases are **additive**, not replacement
- "Similarity" is the key differentiator
- Sets up when to use vector databases
